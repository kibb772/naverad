# 요구사항 문서: 열끈 광고 분석기

## Intent Analysis

- **사용자 요청**: 알려조(allyojo.com)와 동일한 구조의 네이버 검색광고 전용 AI 광고 성과 분석·관리 서비스 구축
- **요청 유형**: New Project (Greenfield)
- **범위**: System-wide (풀스택 웹 애플리케이션)
- **복잡도**: Complex (AI 통합, 외부 API 연동, 실시간 모니터링, 알림 시스템)

---

## 1. 서비스 개요

**서비스명**: 열끈 광고 분석기
**설명**: 네이버 검색광고 전용 AI 기반 광고 성과 자동 분석·관리 서비스. 네이버 검색광고 데이터를 자동 수집·분석하여 이상 탐지, 원인 진단, 개선 제안을 제공하고, 캠페인/광고그룹/키워드를 직접 관리할 수 있는 통합 플랫폼.

---

## 2. 기능 요구사항 (Functional Requirements)

### FR-01: 사용자 인증 및 계정 관리

- **FR-01.1**: 카카오 OAuth 로그인 지원
- **FR-01.2**: 이메일/비밀번호 기반 회원가입 및 로그인
- **FR-01.3**: 회원가입 시 필수 정보 수집 (이메일, 비밀번호, 이름, 전화번호)
- **FR-01.4**: 약관 동의 처리 (서비스 이용약관, 개인정보 처리방침, 광고 데이터 수집 동의, 마케팅 수신 동의)
- **FR-01.5**: 비밀번호 재설정 기능
- **FR-01.6**: 세션 관리 및 자동 로그아웃

### FR-02: 네이버 검색광고 API 연동

- **FR-02.1**: 네이버 검색광고 API 인증 (API 키 + Secret Key + Customer ID)
- **FR-02.2**: 캠페인 데이터 자동 수집 (캠페인/광고그룹/키워드/소재 단위)
- **FR-02.3**: 핵심 지표 수집: 노출수, 클릭수, CTR, CPC, 전환수, 전환율, ROAS, 일예산, 소진액
- **FR-02.4**: 데이터 동기화 주기 설정 (최소 1시간 단위)
- **FR-02.5**: 연동 상태 모니터링 및 오류 알림

### FR-03: 통합 대시보드

- **FR-03.1**: 핵심 지표 실시간 표시 (노출수, 클릭수, CTR, CPC, 전환율, ROAS)
- **FR-03.2**: 기간별 성과 비교 (오늘 vs 어제, 7일 평균, 30일 평균)
- **FR-03.3**: 캠페인별 성과 요약 카드 (지출, ROAS, 변동률)
- **FR-03.4**: 캠페인/광고그룹/키워드 단위 드릴다운 분석
- **FR-03.5**: 성과 추이 차트 (일별/주별/월별)
- **FR-03.6**: 이상 감지 알림 배너 표시

### FR-04: 통합 캠페인 관리 (직접 조절)

- **FR-04.1**: 캠페인 ON/OFF 토글
- **FR-04.2**: 광고그룹 ON/OFF 토글
- **FR-04.3**: 키워드 ON/OFF 토글
- **FR-04.4**: 캠페인 일예산 조정
- **FR-04.5**: 키워드 입찰가 조정
- **FR-04.6**: 새 캠페인 생성 (캠페인명, 일예산, 시작/종료일, 입찰 전략)
- **FR-04.7**: 새 광고그룹 생성
- **FR-04.8**: 새 키워드 추가
- **FR-04.9**: 조작 이력 로깅 (누가, 언제, 무엇을 변경했는지)

### FR-05: AI 이상 탐지

- **FR-05.1**: 핵심 지표(CTR, CPC, 전환율, ROAS, 소진액) 급변 자동 감지
- **FR-05.2**: 기준값 자동 학습 (계정 자체 히스토리 기반 7일/30일 이동 평균)
- **FR-05.3**: 심각도 분류 (주의/경고/위험)
- **FR-05.4**: 이상 감지 시 대시보드 실시간 알림 표시
- **FR-05.5**: 이상 감지 이력 관리 및 조회

### FR-06: AI 원인 진단

- **FR-06.1**: 이상 감지 시 GPT-4.1 기반 자동 원인 분석
- **FR-06.2**: 원인 분류 (키워드 경쟁 심화, 소재 피로도, 시즌 영향, 품질지수 변동 등)
- **FR-06.3**: 각 원인별 기여도(%) 표시
- **FR-06.4**: 자연어 기반 전문가 수준 설명 제공

### FR-07: AI 개선 제안

- **FR-07.1**: 원인 진단 결과 기반 구체적 액션 아이템 제시
- **FR-07.2**: 우선순위별 정렬
- **FR-07.3**: 제안 예시: 입찰가 조정, 키워드 일시중지, 소재 교체, 예산 재배분
- **FR-07.4**: 예상 개선 효과 추정 (가능한 경우)

### FR-08: AI 성과 리포트

- **FR-08.1**: GPT-4.1 기반 일간 성과 요약 (한글 자연어)
- **FR-08.2**: 성과 등급 부여 (매우 좋음/좋음/보통/경고/위험)
- **FR-08.3**: 변화 요인 분석 포함
- **FR-08.4**: 내일 액션 플랜 포함
- **FR-08.5**: 캠페인별 상세 분석 포함

### FR-09: 알림 시스템

- **FR-09.1**: 대시보드 내 실시간 알림 (이상 감지, 시스템 알림)
- **FR-09.2**: 이메일 알림 (이상 감지 즉시 + 일간 리포트)
- **FR-09.3**: 카카오톡 알림톡 (매일 아침 일간 요약 + 이상 감지 즉시)
- **FR-09.4**: 알림 기준 커스터마이징 (어떤 지표, 어떤 임계값에서 알림)
- **FR-09.5**: 알림 채널별 ON/OFF 설정

### FR-10: 랜딩 페이지 및 정적 페이지

- **FR-10.1**: 서비스 소개 랜딩 페이지 (기능 하이라이트, 사용 방법, 후기)
- **FR-10.2**: 서비스 상세 소개 페이지
- **FR-10.3**: 회사 소개 페이지
- **FR-10.4**: 개인정보 처리방침 페이지
- **FR-10.5**: 서비스 이용약관 페이지

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 대시보드 초기 로딩 3초 이내
- API 응답 시간 500ms 이내 (일반 조회)
- AI 진단 응답 시간 30초 이내

### NFR-02: 보안 (SECURITY 확장 규칙 적용)
- SECURITY-01 ~ SECURITY-15 전체 적용
- 네이버 검색광고 API 키 암호화 저장
- 사용자 비밀번호 적응형 해싱 (bcrypt/argon2)
- HTTPS 전용 (TLS 1.2+)
- CSRF/XSS 방어
- Rate limiting (로그인, API 엔드포인트)

### NFR-03: 확장성
- 초기 100명 사용자 기준 설계
- 수평 확장 가능한 아키텍처

### NFR-04: 가용성
- 99.5% 이상 가용성 목표
- 데이터 수집 실패 시 자동 재시도

### NFR-05: 테스팅 (PBT 확장 규칙 적용)
- PBT-01 ~ PBT-10 전체 적용
- 데이터 변환, 직렬화, 비즈니스 로직에 속성 기반 테스트 적용
- fast-check (JavaScript/TypeScript PBT 프레임워크) 사용

### NFR-06: 접근성
- WCAG 2.1 AA 수준 준수 목표
- 키보드 네비게이션 지원
- 스크린 리더 호환

## 4. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js (React) + TypeScript |
| 백엔드 | Node.js + TypeScript |
| AI 엔진 | OpenAI GPT-4.1 |
| 데이터베이스 | PostgreSQL (관계형) + Redis (캐시) |
| 인증 | 카카오 OAuth + JWT |
| 외부 API | 네이버 검색광고 API |
| 알림 | 카카오 알림톡 API + 이메일 (Nodemailer/SES) |
| PBT 프레임워크 | fast-check |
| 테스트 러너 | Vitest |
